---
title: 'Safety checker'
description: 'Geofence validation server and client for enforcing flight boundaries and speed limits.'
---

The safety checker system validates vehicle commands against a geofence configuration before they are sent to the autopilot. It runs as a separate process so it continues to function even if the mission script crashes.

```python
from aerpawlib.v1 import SafetyCheckerServer, SafetyCheckerClient
```

## Architecture

The system uses a ZMQ **REQ/REP** pattern:

- `SafetyCheckerServer` binds a ZMQ REP socket and loops, processing validation requests.
- `SafetyCheckerClient` connects a ZMQ REQ socket and calls validation methods synchronously.

All messages are zlib-compressed JSON.

## SafetyCheckerServer

```python
SafetyCheckerServer(vehicle_config_filename: str, server_port: int = 14580)
```

Loads a YAML configuration file, starts the ZMQ server, and **blocks** until the process is terminated.

<ParamField path="vehicle_config_filename" type="string" required>
  Path to the YAML configuration file (see format below).
</ParamField>

<ParamField path="server_port" type="number" default="14580">
  Port to bind the ZMQ REP socket.
</ParamField>

### YAML configuration format

```yaml
# geofence_config.yaml
vehicle_type: copter      # "copter" or "rover"
max_speed: 15             # maximum allowed speed (m/s)
min_speed: 0.5            # minimum allowed speed (m/s)
max_alt: 120              # maximum altitude, meters (copter only)
min_alt: 5                # minimum altitude, meters (copter only)
include_geofences:        # list of KML files defining allowed areas
  - flight_area.kml
exclude_geofences:        # list of KML files defining no-go zones
  - no_fly_zone.kml
```

KML paths are resolved relative to the directory containing the YAML file.

### Starting the server

<Tabs>
  <Tab title="CLI (recommended)">
    ```bash
    python -m aerpawlib.v1.safety --port 14580 --vehicle_config geofence_config.yaml
    ```
  </Tab>
  <Tab title="Python">
    ```python
    from aerpawlib.v1 import SafetyCheckerServer

    # Constructor blocks; runs until terminated
    SafetyCheckerServer("geofence_config.yaml", server_port=14580)
    ```
  </Tab>
  <Tab title="Background process">
    ```python
    from multiprocessing import Process
    from aerpawlib.v1 import SafetyCheckerServer

    def run_server():
        SafetyCheckerServer("geofence_config.yaml", 14580)

    server = Process(target=run_server, daemon=True)
    server.start()
    ```
  </Tab>
</Tabs>

### Validation logic

For each waypoint command, the server checks (in order):

1. Altitude is within `[min_alt, max_alt]` (copter only).
2. Destination is inside at least one `include_geofence`.
3. Destination is not inside any `exclude_geofence`.
4. The path from current to destination does not cross any `include_geofence` boundary.
5. The path does not enter any `exclude_geofence`.

### KML geofence format

```xml
<?xml version="1.0" encoding="UTF-8"?>
<kml xmlns="http://www.opengis.net/kml/2.2">
  <Document>
    <Placemark>
      <name>Flight Area</name>
      <Polygon>
        <outerBoundaryIs>
          <LinearRing>
            <coordinates>
              -78.700,35.700,0
              -78.600,35.700,0
              -78.600,35.800,0
              -78.700,35.800,0
              -78.700,35.700,0
            </coordinates>
          </LinearRing>
        </outerBoundaryIs>
      </Polygon>
    </Placemark>
  </Document>
</kml>
```

## SafetyCheckerClient

```python
SafetyCheckerClient(
    addr: str,
    port: int,
    timeout_s: float = 10.0,
)
```

<ParamField path="addr" type="string" required>
  IP address of the safety checker server.
</ParamField>

<ParamField path="port" type="number" required>
  Port the server is listening on.
</ParamField>

<ParamField path="timeout_s" type="number" default="10.0">
  Send/receive timeout in seconds. Raises `TimeoutError` if the server does not respond within this time.
</ParamField>

The client can be used as a context manager:

```python
with SafetyCheckerClient("127.0.0.1", 14580) as checker:
    ok, msg = checker.check_server_status()
```

Or managed manually:

```python
checker = SafetyCheckerClient("127.0.0.1", 14580)
try:
    ok, msg = checker.check_server_status()
finally:
    checker.close()
```

### Methods

All methods return `Tuple[bool, str]`: `(True, "")` on success, or `(False, error_message)` on failure.

#### `check_server_status`

```python
checker.check_server_status() -> Tuple[bool, str]
```

Verifies the server is reachable and responding.

```python
ok, msg = checker.check_server_status()
if not ok:
    raise RuntimeError(f"Safety checker offline: {msg}")
```

#### `validate_waypoint_command`

```python
checker.validate_waypoint_command(
    current_location: Coordinate,
    next_location: Coordinate,
) -> Tuple[bool, str]
```

Validates that the path from `current_location` to `next_location` stays inside the geofence and avoids no-go zones. Call this before every `goto_coordinates`.

```python
ok, msg = checker.validate_waypoint_command(drone.position, target)
if ok:
    await drone.goto_coordinates(target)
else:
    print(f"Invalid waypoint: {msg}")
```

#### `validate_change_speed_command`

```python
checker.validate_change_speed_command(new_speed: float) -> Tuple[bool, str]
```

Ensures `new_speed` is within `[min_speed, max_speed]` from the server config.

```python
ok, msg = checker.validate_change_speed_command(12.0)
if ok:
    await drone.set_groundspeed(12.0)
```

#### `validate_takeoff_command`

```python
checker.validate_takeoff_command(
    takeoff_alt: float,
    current_lat: float,
    current_lon: float,
) -> Tuple[bool, str]
```

Validates `takeoff_alt` against the configured altitude limits (copter only). Also records the takeoff location for subsequent landing validation.

```python
ok, msg = checker.validate_takeoff_command(
    10, drone.position.lat, drone.position.lon
)
if ok:
    await drone.takeoff(10)
```

#### `validate_landing_command`

```python
checker.validate_landing_command(
    current_lat: float,
    current_lon: float,
) -> Tuple[bool, str]
```

Ensures the drone is within 5 meters of the takeoff location before landing. Requires `validate_takeoff_command` to have been called first.

```python
ok, msg = checker.validate_landing_command(
    drone.position.lat, drone.position.lon
)
if ok:
    await drone.land()
else:
    print(f"Cannot land here: {msg}")
```

## Full integration example

```python
from aerpawlib.v1 import (
    Drone, Coordinate, BasicRunner, entrypoint,
    SafetyCheckerClient,
)

class SafeMission(BasicRunner):
    def __init__(self):
        self.checker = SafetyCheckerClient("127.0.0.1", 14580)

    def cleanup(self):
        self.checker.close()

    @entrypoint
    async def run(self, drone: Drone):
        ok, msg = self.checker.check_server_status()
        if not ok:
            print(f"Safety checker not running: {msg}")
            return

        ok, msg = self.checker.validate_takeoff_command(
            10, drone.position.lat, drone.position.lon
        )
        if not ok:
            print(f"Cannot take off: {msg}")
            return
        await drone.takeoff(10)

        waypoints = [
            Coordinate(35.7275, -78.6960, 10),
            Coordinate(35.7280, -78.6955, 15),
        ]
        for wp in waypoints:
            ok, msg = self.checker.validate_waypoint_command(drone.position, wp)
            if ok:
                await drone.goto_coordinates(wp)
            else:
                print(f"Skipping {wp}: {msg}")
                break

        ok, msg = self.checker.validate_landing_command(
            drone.position.lat, drone.position.lon
        )
        if ok:
            await drone.land()
        else:
            print(f"Landing validation failed: {msg}")
            await drone.return_to_launch()
```
