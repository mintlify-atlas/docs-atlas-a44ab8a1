This guide takes you from zero to a flying drone mission. You will write a script, connect to a simulated vehicle, and watch it execute a waypoint flight.

## Prerequisites

- aerpawlib installed — see [Installation](/installation)
- ArduPilot SITL running (install with `aerpawlib-setup-sitl` if needed)

## Steps

<Steps>
  <Step title="Start ArduPilot SITL">
    In a separate terminal, launch the Copter simulator:

    ```bash
    sim_vehicle.py -v ArduCopter --out=udp:127.0.0.1:14550 --console
    ```

    Wait until the console shows `GPS lock` and `EKF ok` before continuing.
  </Step>

  <Step title="Write your mission script">
    Create a file called `my_mission.py`.

    <Tabs>
      <Tab title="v1 API">
        ```python my_mission.py
        from aerpawlib.v1 import Drone, Coordinate, BasicRunner, entrypoint

        class MyMission(BasicRunner):
            @entrypoint
            async def run(self, vehicle: Drone):
                await vehicle.takeoff(10)
                for wp in [
                    Coordinate(35.7275, -78.6960, 10),
                    Coordinate(35.7280, -78.6955, 10),
                ]:
                    await vehicle.goto_coordinates(wp)
                await vehicle.land()
        ```

        The `@entrypoint` decorator marks the single method that `BasicRunner` calls when the script starts. `vehicle.takeoff(alt)`, `goto_coordinates(coord)`, and `land()` are all awaitable — execution pauses until each maneuver completes.
      </Tab>
      <Tab title="v2 API">
        ```python my_mission.py
        from aerpawlib.v2 import Drone, Coordinate, VectorNED, BasicRunner, entrypoint

        class MyMission(BasicRunner):
            @entrypoint
            async def run(self, drone: Drone):
                await drone.takeoff(altitude=10)
                await drone.goto_coordinates(drone.position + VectorNED(20, 0))
                await drone.land()
        ```

        The v2 API is async-first with a single event loop. Use `VectorNED(north, east)` to compute target coordinates relative to the current position, or pass an absolute `Coordinate`.
      </Tab>
    </Tabs>
  </Step>

  <Step title="Run the mission">
    <Tabs>
      <Tab title="v1 API">
        ```bash
        aerpawlib --script my_mission.py \
          --conn udpin://127.0.0.1:14550 \
          --vehicle drone
        ```
      </Tab>
      <Tab title="v2 API">
        ```bash
        aerpawlib --script my_mission.py \
          --conn udpin://127.0.0.1:14550 \
          --vehicle drone \
          --api-version v2 \
          --no-aerpaw-environment
        ```
      </Tab>
    </Tabs>

    aerpawlib discovers your `BasicRunner` subclass automatically, connects to the vehicle, waits for it to become armable, then executes `run()`.
  </Step>

  <Step title="Watch it fly">
    You should see log output like:

    ```
    [INFO] Found runner class: MyMission
    [INFO] Connecting to vehicle...
    [INFO] BasicRunner: starting entrypoint 'run'
    ```

    The drone takes off to 10 m, navigates to each waypoint, and lands. After the mission completes, aerpawlib exits with code 0.
  </Step>
</Steps>

## How it works

When you run `aerpawlib --script my_mission.py`, the runner:

1. Imports your script and scans for a class that directly subclasses `BasicRunner`, `StateMachine`, or `ZmqStateMachine`.
2. Connects to the vehicle at the address given by `--conn`.
3. Waits for the vehicle to become armable (in SITL this happens automatically; on AERPAW hardware a safety pilot must arm the vehicle).
4. Calls your `run()` method and awaits its completion.
5. If the vehicle is still armed when `run()` returns and `--skip-rtl` was not passed, it automatically returns to launch.
