---
title: "v2 API overview"
description: "Modern async-first API for controlling drones and ground vehicles on the AERPAW platform."
---

v2 is a complete rewrite of the aerpawlib vehicle control API. It runs on a single asyncio event loop, streams telemetry through native async subscriptions, and provides descriptor-based decorators for defining runners.

## Key improvements over v1

| Area | v1 | v2 |
|------|-----|-----|
| Event loop | Dual-loop (separate MAVSDK thread) | Single asyncio loop — all commands use direct `await` |
| Telemetry | `ThreadSafeValue` shared state | Plain attributes updated by subscription callbacks |
| Runners | Function-attribute pattern | Descriptor-based decorators or config dataclasses |
| Non-blocking commands | Not supported | `VehicleTask` with `progress`, `cancel()`, `wait_done()` |
| Safety | External | Built-in `can_takeoff`, `can_goto`, `can_land` with optional `SafetyCheckerClient` |
| Connection handling | Manual | `ConnectionHandler` with heartbeat monitoring |

## Quick start

```python
from aerpawlib.v2 import BasicRunner, Drone, VectorNED, entrypoint

class MyMission(BasicRunner):
    @entrypoint
    async def run(self, drone: Drone):
        await drone.takeoff(altitude=10)
        await drone.goto_coordinates(drone.position + VectorNED(20, 0))
        await drone.land()
```

Run with:

```bash
aerpawlib --api-version v2 --script my_mission.py --vehicle drone --conn udpin://127.0.0.1:14550
```

## Import patterns

All public symbols are available from the top-level `aerpawlib.v2` package:

```python
from aerpawlib.v2 import (
    Drone, Rover, Vehicle, DummyVehicle, VehicleTask,
    BasicRunner, BasicRunnerConfig,
    StateMachine, StateMachineConfig, StateSpec,
    ZmqStateMachine, ZmqStateMachineConfig,
    entrypoint, state, timed_state, background, at_init,
    expose_zmq, expose_field_zmq,
    Coordinate, VectorNED, Battery, GPSInfo, Attitude,
    Waypoint, read_from_plan, read_from_plan_complete, get_location_from_waypoint,
    read_geofence, inside, do_intersect,
    ExternalProcess,
    run_zmq_proxy, check_zmq_proxy_reachable,
    MockVehicle,
    VehicleProtocol, GPSProtocol,
    AERPAW_Platform,
)
```

Safety types live in the sub-package:

```python
from aerpawlib.v2.safety import SafetyCheckerClient, NoOpSafetyChecker
```
