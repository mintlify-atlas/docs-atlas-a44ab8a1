---
title: "State machine missions"
description: "Build multi-phase missions with @state, @timed_state, and @background decorators."
---

When a mission has distinct phases, conditional transitions, or parallel logging tasks, use `StateMachine` instead of `BasicRunner`. Each phase is a decorated method called a _state_. States return the name of the next state, or `None` to stop the machine.

## The StateMachine pattern

```python
from aerpawlib.v1 import Drone, StateMachine, state

class MyMission(StateMachine):
    @state(name="takeoff", first=True)
    async def takeoff_state(self, vehicle: Drone):
        await vehicle.takeoff(10)
        return "patrol"

    @state(name="patrol")
    async def patrol_state(self, vehicle: Drone):
        await vehicle.goto_coordinates(next_point)
        return "land"

    @state(name="land")
    async def land_state(self, vehicle: Drone):
        await vehicle.land()
        return None  # None stops the machine
```

Rules:
- Exactly one state must have `first=True`. This is the initial state.
- Return the `name` string of the next state to transition to it.
- Return `None` (or nothing) to stop the machine.
- State names must be non-empty strings and unique within a class.

## Decorators

### @state

```python
@state(name="my_state", first=False)
async def some_state(self, vehicle):
    # ... do work ...
    return "next_state"
```

| Argument | Type | Description |
|---|---|---|
| `name` | `str` | Unique state identifier used for transitions |
| `first` | `bool` | If `True`, this is the initial state (default: `False`) |

### @timed_state

A timed state runs for a fixed duration before advancing. Optionally calls the function repeatedly (looping) within that duration.

```python
from aerpawlib.v1 import timed_state

@timed_state(name="hover", duration=5.0, loop=False, first=False)
async def hover_state(self, vehicle):
    # Called once (or every tick if loop=True) then waits for duration seconds
    return "next_state"
```

| Argument | Type | Description |
|---|---|---|
| `name` | `str` | Unique state identifier |
| `duration` | `float` | Minimum seconds to remain in this state |
| `loop` | `bool` | If `True`, re-invoke the function every state-machine tick within the duration |
| `first` | `bool` | If `True`, this is the initial state |

### @background

Background tasks run concurrently with the state machine for the duration of the entire mission. They are started before the first state and cancelled when the machine stops.

```python
from aerpawlib.v1 import background
import asyncio

@background
async def log_position(self, vehicle):
    while True:
        print(f"Position: {vehicle.position}")
        await asyncio.sleep(1)
```

<Note>
  Background tasks are automatically restarted if they raise an exception. In v2, the retry is bounded to 10 consecutive failures before the task is stopped permanently.
</Note>

### @at_init

Init tasks run once before the vehicle is armed, useful for setup work like reading config files or making initial measurements:

```python
from aerpawlib.v1 import at_init

@at_init
async def setup(self, vehicle):
    print("Running pre-arm initialization")
```

## Complete example: square flight with logging

The following is the full `examples/v1/squareoff_logging.py` from the repository. It demonstrates a `StateMachine` with multiple states, a `@timed_state`, and a `@background` task that logs vehicle data to CSV.

```python examples/v1/squareoff_logging.py
from argparse import ArgumentParser
import asyncio
import csv
import datetime
import time
from typing import List, TextIO
import os

from aerpawlib.v1.runner import StateMachine, state, background, timed_state
from aerpawlib.v1.util import VectorNED
from aerpawlib.v1.vehicle import Drone, Rover, Vehicle

FLIGHT_ALT = 5  # m
SQUARE_SIZE = 10  # m
LOCATION_TOLERANCE = 2  # m
WAIT_TIME = 5  # s
LEG_VELOCITY = 5  # m/s


class SquareOff(StateMachine):
    _next_sample: float = 0
    _sampling_delay: float
    _cur_line: int
    _csv_writer: object
    _log_file: TextIO

    def initialize_args(self, extra_args: List[str]):
        default_file = f"GPS_DATA_{datetime.datetime.now().strftime('%Y-%m-%d_%H:%M:%S')}.csv"

        parser = ArgumentParser()
        parser.add_argument("--output", required=False, default=default_file)
        parser.add_argument("--samplerate", required=False, default=1, type=float)
        args = parser.parse_args(args=extra_args)

        self._sampling_delay = 1 / args.samplerate
        self._log_file = open(args.output, "a+")
        self._log_file.seek(0)
        self._cur_line = sum(1 for _ in self._log_file) + 1
        self._log_file.seek(0, 2)
        self._csv_writer = csv.writer(self._log_file)

    @background
    async def periodic_dump(self, vehicle: Vehicle):
        # Periodically log vehicle status to CSV
        self._next_sample = time.time() + self._sampling_delay
        pos = vehicle.position
        self._csv_writer.writerow([self._cur_line, pos.lon, pos.lat, pos.alt])
        self._log_file.flush()
        os.fsync(self._log_file.fileno())
        self._cur_line += 1
        await asyncio.sleep(self._sampling_delay)

    def cleanup(self):
        self._log_file.close()

    _legs = ["leg_north", "leg_west", "leg_south", "leg_east"]
    _current_leg = 0

    @state(name="start", first=True)
    async def start(self, vehicle: Vehicle):
        if isinstance(vehicle, Drone):
            return "take_off"
        elif isinstance(vehicle, Rover):
            return self._legs[self._current_leg]
        raise Exception("Vehicle not supported")

    @state(name="take_off")
    async def take_off(self, drone: Drone):
        await drone.takeoff(FLIGHT_ALT)
        await drone.set_groundspeed(LEG_VELOCITY)
        return "leg_north"

    @timed_state(name="at_position", duration=WAIT_TIME)
    async def at_position(self, _):
        self._current_leg += 1
        if self._current_leg < len(self._legs):
            return self._legs[self._current_leg]
        return "finish"

    async def command_leg(self, vehicle: Vehicle, dNorth: float, dEast: float):
        await vehicle.goto_coordinates(
            vehicle.position + VectorNED(dNorth, dEast),
            tolerance=LOCATION_TOLERANCE,
        )

    @state(name="leg_north")
    async def leg_north(self, vehicle: Vehicle):
        await self.command_leg(vehicle, SQUARE_SIZE, 0)
        return "at_position"

    @state(name="leg_west")
    async def leg_west(self, vehicle: Vehicle):
        await self.command_leg(vehicle, 0, -SQUARE_SIZE)
        return "at_position"

    @state(name="leg_south")
    async def leg_south(self, vehicle: Vehicle):
        await self.command_leg(vehicle, -SQUARE_SIZE, 0)
        return "at_position"

    @state(name="leg_east")
    async def leg_east(self, vehicle: Vehicle):
        await self.command_leg(vehicle, 0, SQUARE_SIZE)
        return "at_position"

    @state(name="finish")
    async def finish(self, vehicle: Vehicle):
        if isinstance(vehicle, Drone):
            return "land"
        elif isinstance(vehicle, Rover):
            print("done!")

    @state(name="land")
    async def land(self, drone: Drone):
        await drone.land()
        print("done!")
```

Run it with optional custom arguments:

```bash
aerpawlib --script examples/v1/squareoff_logging.py \
  --conn udp://127.0.0.1:14550 \
  --vehicle drone \
  --output my_log.csv \
  --samplerate 2
```

## State diagram for squareoff_logging

The state transitions for the example above:

```
start (first=True)
  ├── drone  → take_off
  └── rover  → leg_north

take_off → leg_north

leg_north → at_position
leg_west  → at_position
leg_south → at_position
leg_east  → at_position

at_position (timed, 5s)
  ├── more legs → next leg state
  └── done      → finish

finish
  ├── drone → land
  └── rover → None (stop)

land → None (stop)
```

## Stopping the machine

You have three equivalent ways to stop a `StateMachine`:

1. **Return `None`** from a state: `return None`
2. **Return nothing** from a state (implicit `None`)
3. **Call `self.stop()`** to stop after the current state completes

<Warning>
  Do not call `self.stop()` and return a state name in the same state function — `stop()` sets `_running = False`, which takes effect at the end of the current state loop tick regardless of the returned name.
</Warning>

## Holding internal state

You can store data between states as instance attributes:

```python
class MyMission(StateMachine):
    _waypoint_index: int = 0
    _waypoints = [Coordinate(35.72, -78.69, 10), Coordinate(35.73, -78.68, 10)]

    @state(name="next", first=True)
    async def next_waypoint(self, vehicle):
        if self._waypoint_index >= len(self._waypoints):
            return "land"
        await vehicle.goto_coordinates(self._waypoints[self._waypoint_index])
        self._waypoint_index += 1
        return "next"

    @state(name="land")
    async def land(self, vehicle):
        await vehicle.land()
```
